# Profile Viewer

Utility to read and visualize JSON files generated by the ZiskEmu profiler.

## Features

- Reads JSON files generated by `CallPathProfiler::save_to_file()`
- Supports compressed (.json.gz) and uncompressed (.json) files
- Shows information for each sample: function, weight, timeDelta, and accumulated time
- Can display the complete call stack for each sample
- Generates profiling summary statistics
- Supports multiple output formats (text, CSV)

## Building

```bash
cd tools/profile-viewer
cargo build --release
```

## Usage

### Basic Usage

```bash
# View all samples
cargo run --release -- profile.json.gz

# View only the first 100 samples
cargo run --release -- profile.json.gz -n 100

# Show the complete call stack for each sample
cargo run --release -- profile.json.gz -s

# Export to CSV
cargo run --release -- profile.json.gz -f csv > samples.csv

# Combine options
cargo run --release -- profile.json.gz -n 50 -s -f text
```

### Options

- `FILE`: Path to the profile file (.json or .json.gz)
- `-n, --limit <N>`: Show only the first N samples (0 = show all) [default: 0]
- `-s, --show-stack`: Show the complete call stack for each sample
- `-f, --format <FORMAT>`: Output format (text, csv) [default: text]

## Output Format

**Note:** Function names are displayed with the format `name #<idx>`, where `<idx>` is the index in the profile's string array. This facilitates correlation with the original JSON file.

### Text Mode (default)

```
======================================================================================================
ZiskEmu Profile Samples
======================================================================================================

Sample       Weight    TimeDelta      CumTime  Function
----------------------------------------------------------------------------------------------------
     0        1.000      0.000000      0.000000  main #5
     1       12.345      0.001234      0.001234  process_instruction #23
     2        5.678      0.000567      0.001801  decode_opcode #45
...
```

### CSV Mode

```
Sample,Weight,TimeDelta,CumulativeTime,Function
0,1.000,0.000000,0.000000,"main #5"
1,12.345,0.001234,0.001234,"process_instruction #23"
2,5.678,0.000567,0.001801,"decode_opcode #45"
...
```

### With Call Stack (-s)

```
     0        1.000      0.000000      0.000000  process_block #67
       Call stack:
        0: main #5
        1: execute_program #12
        2: process_block #67
```

## Summary Statistics

At the end of execution, general statistics are displayed:

- Total samples
- Total weight and total time
- Weight statistics (average, minimum, maximum, median)
- Top 5 functions by accumulated weight

## Complete Example

```bash
# Generate a profile from the emulator
cd ../../
cargo run --release --bin ziskemu -- execute program.elf

# This generates profile.json.gz

# View the first 20 samples with call stacks
cd tools/profile-viewer
cargo run --release -- ../../profile.json.gz -n 20 -s
```

## Integration

This tool reads the Firefox Profiler format generated by `CallPathProfiler::save_to_file()` in the `emulator/src/stats/profiler.rs` module.

The files can also be visualized at https://profiler.firefox.com by loading the JSON directly.
